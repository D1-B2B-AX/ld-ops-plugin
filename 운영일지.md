## 🚫 절대 규칙 — Python 스크립트 우회 금지

v0.5 실측에서 Claude가 `build_matrix.py`·`generate_ops_md.py` 등 여러 스크립트를 **우회하고 JSON/MD를 직접 작성**해 스키마가 깨진 사고 있음. v1.0은 다음 강제 규칙:

| 단계 | 반드시 호출 | Write 도구 |
|---|---|---|
| Step 0.5 일정 조립 | `python scripts/compose_schedule.py ...` | ❌ sessions.json 수동 금지 |
| Step 1 격자 구축 | `python scripts/build_matrix.py ...` | ❌ arriving_cells.json 수동 금지 |
| Step 2 증거 수집 | `python scripts/collect_evidence.py ...` | ❌ evidence.json 수동 금지 |
| Step 3 분류 입력 | `python scripts/classify_evidence.py ...` | ❌ classified_cells.json 수동 금지 |
| Step 3/0.5 LLM 병합 | `python scripts/apply_llm_responses.py ...` | ❌ 직접 파일 수정 금지 |
| Step 4 렌더 | `python scripts/generate_ops_md.py ...` | ❌ MD 수동 작성 금지 |
| Step 4.6 검증 | `python scripts/verify_output_format.py --strict` | ❌ |
| Step 4c-2 슬랙 | `python scripts/md_to_slack.py ...` | ❌ .slack.txt 수동 금지 |

### Claude 역할 한정 (LLM 필요 지점만)

- **Step 0** — MCP 도구로 5소스 데이터 수집. 결과를 `runtime/s{1..5}_*.json`에 `{deal_id: result}` 형식으로 저장
- **Step 0.5 후속** — `pending_slack_updates`의 `llm_prompt`를 LLM 처리해 `slack_responses.json` 생성 → `apply_llm_responses.py --mode slack` 실행
- **Step 3 후속** — `classified_cells.json`의 PENDING 셀 `llm_prompt`를 LLM 처리해 `label_responses.json` 생성 → `apply_llm_responses.py --mode labels` 실행
- **Step 5** — 사용자 자연어 피드백 파싱 → 조작 명령 JSON → `apply_feedback.py` 실행

### 검증 게이트

- Step 4.6 (`verify_output_format.py --strict`) 실패 시 **Step 4 재실행**. 실패 99%는 Claude Write 도구 수동 작성.

## 📊 진척도 표시 (사용자 친화적 5단계)

TaskCreate·TaskUpdate는 **사용자용 5단계**로만 생성. 내부 Step 세부(0.5·4.6 등)는 사용자에 노출 X.

**사용자 화면 체크리스트 (정확히 이 5개):**

```
◻ 1. 환경 준비
◻ 2. 교육 데이터 모으기
◻ 3. 오늘 체크 항목 선별 + 증거 확인
◻ 4. 운영일지 작성
◻ 5. 자연어 피드백 대기
```

**내부 Step ↔ 사용자 5단계:**

| 사용자 단계 | 내부 Step |
|---|---|
| 1. 환경 준비 | MCP 인증 확인, DB 최신화 |
| 2. 교육 데이터 모으기 | Step 0 (5소스 수집), Step 0.5 (일정 조립·슬랙 PENDING 처리) |
| 3. 오늘 체크 항목 선별 + 증거 확인 | Step 1 (격자), Step 2 (증거), Step 3 (LLM 분류·PENDING 처리) |
| 4. 운영일지 작성 | Step 4 (렌더), Step 4.6 (검증), 슬랙 변환 |
| 5. 자연어 피드백 대기 | Step 5 (사용자 입력 대기) |

라벨 작성 원칙: 기술 용어 지양, 한글 자연어 친화.

## 실행 흐름 요약

### 1) 환경 준비 (Claude)
```bash
python <plugin-root>/scripts/check_env.py \
  --settings <plugin-root>/config/settings.json
```
DB 자동 다운로드 + MCP 인증 확인.

### 2) Step 0 — 5/6소스 MCP 수집 (Claude)
`data-skills/1~5` 병렬 호출 (한 `function_calls` 블록 내 동시). 결과는 **`{deal_id: result}` 딕셔너리**로 묶어 저장:
- `runtime/s1_deals.json` (세일즈맵 — 이미 배열 형태)
- `runtime/s2_calendar.json`, `s3_slack.json`, `s4_gmail.json`, `s5_drive.json`

**(선택) 스킬 6 — 노션 강의 캘린더 (2팀 전용, v1.0 MVP):**
- `owner.team`이 "교육 2팀" 또는 "교육 2팀 운영 파트"일 때만
- 통합 권한 확보 전까지 **수동 CSV 경로**:
  1. 노션 페이지 → `...` → 내보내기 → "Markdown & CSV"
  2. CSV 파일을 `runtime/notion_calendar.csv`로 복사
  3. `run_pipeline.sh pre`가 자동으로 `parse_notion_csv.py` 실행해 `runtime/s6_notion.json` 생성

### 3) Step 0.5~3 자동 실행 (pre)
```bash
cd <plugin-root>/ops-plugin
bash run_pipeline.sh pre
# PowerShell: .\run_pipeline.ps1 -Stage pre
```

### 4) PENDING 처리 (Claude + LLM)

**Step 0.5 슬랙 PENDING:**
- `sessions.json`의 `pending_slack_updates` 각 항목 `llm_prompt`를 LLM에 전달
- 응답 `{"sessions": [...], "no_schedule_info": bool}` 수집
- `runtime/slack_responses.json` 저장
- `python scripts/apply_llm_responses.py --mode slack --target runtime/sessions.json --responses runtime/slack_responses.json`

**Step 3 라벨 PENDING:**
- `classified_cells.json`의 label="PENDING" 셀 각 llm_prompt를 LLM에 전달
- 응답 `{"label": "이모지", "confidence": 0~1}` 수집
- `runtime/label_responses.json` 저장 (키: 셀 인덱스 or `"deal_id:checkpoint_id[:session_no]"`)
- `python scripts/apply_llm_responses.py --mode labels --target runtime/classified_cells.json --responses runtime/label_responses.json`

### 5) Step 4~4.6 자동 실행 (post)
```bash
bash run_pipeline.sh post
# PowerShell: .\run_pipeline.ps1 -Stage post
```

### 6) Step 5 — 자연어 피드백 루프 (선택)
사용자 자연어 입력 → LLM 파싱 → 조작 명령 JSON → 
```bash
python scripts/apply_feedback.py --state state/ops_state.json --commands commands.json
```

**v0 MVP 조작 4종:** 셀 상태 override / 일정 이동 / watchlist 제외 / deal_flag 설정
